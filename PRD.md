**작성일**: 2026-05-29
**작성자**: sangwoopark@com2us.com
**버전**: 0.1 (Draft)

---

## 1. 제품 개요 (Overview)

비디오 파일을 드래그 앤 드롭하면, 해당 비디오의 모든 프레임을 JPG 이미지로 추출해주는 macOS 데스크톱 앱.

### 1.1 한 줄 설명
**"비디오를 끌어놓으면, 같은 폴더에 프레임 이미지를 자동으로 정리해주는 가장 단순한 Mac 앱."**

### 1.2 해결하려는 문제
- 영상 편집/분석/머신러닝 데이터셋 준비 시, 프레임 단위 이미지 추출이 자주 필요함.
- FFmpeg CLI는 진입장벽이 있고, 매번 명령어를 외우거나 검색해야 함.
- GUI 도구는 대부분 무겁거나, 사용자가 원하는 단순한 워크플로우를 지원하지 않음.

### 1.3 목표 사용자
- 영상 콘텐츠 제작자, 썸네일·스토리보드 제작자
- 머신러닝 데이터셋 준비가 필요한 개발자/연구자
- 영상 분석이 필요한 일반 사용자

## 2. 핵심 사용자 플로우 (User Flow)

```
[1] 앱 실행
       ↓
[2] 메인 창에 비디오 파일 드래그 앤 드롭
       ↓
[3] JPG 품질 설정 다이얼로그 (1~100, 기본값 90)
       ↓ (확인)
[4] 진행률 표시 (현재 N / 총 M 프레임)
       ↓
[5] 완료 → 결과 폴더 자동 오픈(옵션)
```

### 2.1 결과 파일 구조

비디오 경로: `/Users/me/Movies/sample.mp4`

변환 후:
```
/Users/me/Movies/
├── sample.mp4
└── sample/                  ← 비디오 이름과 동일한 폴더 자동 생성
    ├── sample_000001.jpg
    ├── sample_000002.jpg
    ├── sample_000003.jpg
    └── ...
```

- 폴더가 이미 존재하면: 사용자에게 `덮어쓰기` / `다른 이름으로 (sample_2)` / `취소` 선택지 제공
- 파일명 규칙: `{비디오파일명}_{6자리 0패딩 인덱스}.jpg` (예: `sample_000001.jpg`)
  - 비디오 파일명에서 확장자는 제외
  - 파일명에 공백/특수문자가 있어도 그대로 사용

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 MVP (필수 기능)

| ID  | 기능 | 설명 |
|-----|------|------|
| F1  | 드래그 앤 드롭 입력 | 메인 창에 비디오 파일을 끌어놓으면 인식 |
| F2  | 파일 선택 버튼 | 드래그 외에 "파일 선택" 버튼도 제공 (보조) |
| F3  | 비디오 메타데이터 표시 | 파일명, 길이, fps, 총 프레임 수, 해상도 미리보기 |
| F4  | JPG 품질 설정 다이얼로그 | 1~100 슬라이더, 기본값 **90**, "기억하기" 체크박스 |
| F5  | 모든 프레임 추출 | 비디오의 실제 fps 그대로 모든 프레임을 JPG로 |
| F6  | 출력 폴더 자동 생성 | 비디오와 같은 디렉토리에 비디오 이름 폴더 생성 |
| F7  | 진행률 표시 | 진행률 바 + "현재 N / 총 M 프레임" 텍스트 |
| F8  | 취소 버튼 | 변환 중에 사용자가 중단 가능 |
| F9  | 완료 알림 | 완료 시 macOS 알림 + "폴더 열기" 버튼 |
| F10 | 폴더 충돌 처리 | 동일 이름 폴더 존재 시 덮어쓰기/이름 변경/취소 선택 |

### 3.2 지원 입력 포맷
- `.mp4`, `.mov`, `.m4v`, `.avi`, `.mkv`, `.webm`, `.flv`, `.wmv`
- (FFmpeg가 지원하는 모든 포맷을 기본적으로 수용)

### 3.3 비기능 요구사항 (Non-Functional)
- **성능**: FullHD 1분 영상(약 1,800프레임) 기준 30초 이내 변환 (M1 이상 기준)
- **안정성**: 변환 중 앱이 멈추지 않도록 추출 작업은 백그라운드 스레드에서 실행
- **디스크 안전**: 변환 전 예상 디스크 사용량 표시 (선택 기능)
- **설정 영속화**: 마지막 사용한 JPG 품질 값 저장

### 3.4 향후 확장 (v0.2 이후, 이번 MVP에는 포함하지 않음)
- 추출 간격 조절 (1초당 1프레임, N프레임마다 1장 등)
- 해상도 다운스케일 옵션
- 여러 비디오 일괄 처리 (batch)
- 시작/종료 시간 지정 (구간 추출)
- PNG/WebP 출력 포맷

## 5. 기술 스택 (Tech Stack)

| 영역 | 선택 | 비고 |
|------|------|------|
| 언어 | **Python 3.11+** | |
| GUI 프레임워크 | **PyQt6** (또는 Tkinter) | 드래그 앤 드롭 지원과 디자인 자유도 면에서 PyQt6 추천 |
| 비디오 처리 | **FFmpeg** (CLI 호출) | 가장 호환성 좋고 빠름. `imageio-ffmpeg`로 번들링 가능 |
| 보조 라이브러리 | `opencv-python` (메타데이터 추출용, 선택) | |
| 패키징 | **py2app** 또는 **PyInstaller** | 최종 `.app` 번들 생성 |
| 설정 저장 | macOS `~/Library/Application Support/VideoFrames/config.json` | |

### 5.1 핵심 명령 (FFmpeg)
```bash
ffmpeg -i "sample.mp4" -q:v {jpg_quality_to_ffmpeg_qscale} \
       "sample/sample_%06d.jpg"
```
- JPG 품질 1~100 → FFmpeg `-q:v` 2~31 매핑 (반비례, 2가 최고품질)
- 진행률 파싱: stderr의 `frame=N` 라인 실시간 파싱

## 6. 엣지 케이스 & 에러 처리

| 상황 | 동작 |
|------|------|
| 비디오 파일이 아닌 파일 드래그 | "지원하지 않는 파일 형식입니다" 토스트 |
| 동일 이름 폴더 이미 존재 | 덮어쓰기 / 이름변경(sample_2) / 취소 다이얼로그 |
| 디스크 공간 부족 | 변환 시작 전 경고, 추출 중 부족 시 중단 + 알림 |
| 손상된 비디오 파일 | FFmpeg 오류 메시지를 사용자 친화적으로 표시 |
| 변환 중 앱 강제 종료 | 다음 실행 시 부분 결과 폴더에 대한 안내 (선택) |
| 매우 긴 영상 (1시간+) | 시작 전 예상 시간/용량 강조 표시 |
| 비디오 파일 경로에 한글/공백 | 정상 처리 (FFmpeg 인자 quoting 주의) |
